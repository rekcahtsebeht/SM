# UML简单介绍(十八)——组件图的基本概念与实例解析
**1、组件图**

组件图又被成为构件图，主要用于静态建模，是表示构件类型的组织以及各种构件之间依赖关系的图。组件图通过对组件间依赖关系的描述来估
件的修改给系统可能带来的影响。

**2、事务**

可替换的物理部分包括软件代码、脚本或命令行文件，也可以表示运行时的对象，文档，数据库等。节点( node )是运行时的物理对象，代表一
